## Missing Constraints

- **Review-gate naming.** No mention that A6_REVIEW (the docs encode XP/role/entitlement/backend-ownership behavior) and A8_OUTPUT_CHECKER must run before readiness, as required by `implementation/AGENTS.md`.
- **`.gitignore` strategy.** No plan to add ignore rules for Flutter (`build/`, `.dart_tool/`) and Functions (`node_modules/`, `lib/`, debug logs) before the first scaffold commit — the main churn source.
- **Default-deny rules posture.** Even though production rules are deferred, Phase 1 should commit to a default-deny Firestore/Storage rules baseline rather than leaving rules absent/open in the emulator scaffold.
- **Secrets/keys handling.** No constraint that Maps and LLM API keys must never be committed (relevant once Maps/AI enter scope); use config/emulator env, not source.

- **Version pinning.** Flutter SDK and Functions Node runtime are raised as questions but no pinned default/lockfile commitment is stated.
